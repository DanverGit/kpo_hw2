# kpo_hw2

	Релизовано
Основной функционал, аналитика

	Паттерны GoF
Singleton - см. DIConatiner.cs 

Abstract factory - BankFactory создает объекты доменной модели.

Facade - используются фасады для взаимодействия с объектами.

Command - команды оборачивают функции и фасады.

	SOLID, GRASP
SRP - у каждого класса своя четкая задача.

OCP - классы легко расширять.

LSP - родителей заменяют наследники.

ISP - интерфейсы реализуют только необходимые методы и свойства.

DIP - работа с зависимостями происходит через DIContainer.

Low coupling - классы связаны слабо.

High cohesion - классы спроектированы под конкретные цели.
